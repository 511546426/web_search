# 情侣记录站 · 设计方案

## 一、选型建议：网页 vs App

**推荐：网页 + PWA（渐进式 Web 应用）**

| 对比       | 网页 + PWA     | 原生 App        |
|------------|----------------|-----------------|
| 对象使用   | 发链接即可打开 | 需下载、更新   |
| 跨设备     | 手机/平板/电脑通用 | 需各平台分别开发 |
| 分享       | 一个链接搞定   | 应用商店分发   |
| 维护       | 改一次全生效   | 多端发版       |
| 浪漫感     | 可做成 PWA「添加到主屏幕」像 App | 体验一致 |

**结论**：用网页实现，并支持 PWA。对象收到链接点开就能看，需要时还可「添加到主屏幕」，像 App 一样用。

## 二、功能规划

1. **时光轴**：按时间线展示在一起的点点滴滴（文字、图片、日期）
2. **纪念日**：在一起的日子、第一次约会等，自动显示倒计时/已过天数
3. **相册**：上传照片，支持简单描述与日期
4. **悄悄话**：写给对方的短句，可设为「仅对方可见」或公开
5. **今日·我们的历史**：当天在过去几年发生过什么（如「一年前的今天我们一起…」）


## 四、数据模型（核心表）

- **memories**：一条记忆（内容、类型、日期、关联图片等）
- **anniversaries**：纪念日（名称、日期、是否每年重复）
- **photos**：相册条目（图片路径、描述、日期）
- **notes**：悄悄话（内容、创建时间、可见性）

---

## 五、浪漫细节建议

- 主题色：暖粉/玫瑰金/浅紫，搭配柔和白/米色背景
- 字体：圆润、易读的中文 + 英文
- 动效：页面切换、卡片出现用轻量动画
- 文案：节日/纪念日自动祝福语、「我们的第 N 天」等
- 背景：可选渐变或极淡的图案/心形点缀（不抢内容）
